🎯 Purpose
🗺️
Single Source of Truth
All 97 KPIs and their version applicability in one place — no more hunting through config files or spreadsheets.
⚙️
Developer Configuration Aid
Instantly see which versions (WP, TY, LY, LLY, BDGT, FCST) each KPI must be wired to during implementation or QA.
🏷️
Type Enforcement Guide
Enforces naming conventions: _r = Retail $, _c = Cost $, _p = %, etc.
🔗
Pair Consistency Rules
Validates that LY = LLY and BDGT = FCST always carry the same KPI set — mismatches surface immediately.
🔍
Deviation Detection
Highlights KPIs that are intentionally absent from certain versions so developers know what not to configure.
📤
Exportable & Shareable
Export any filtered view as CSV from the KPI Pivot tab for offline review or ticket attachments.
Note for developers: Unless a KPI appears in the deviation sections of the Sanity Check tab, assume it is applicable to all 6 versions. Deviations are intentional and documented — do not add missing KPIs without confirming with the product team.
🏷️ KPI Naming Convention — Suffix → Type Rules
_r→dollarsRetailTwoDecimals
_c→dollarsCostTwoDecimals
_p→percentageTwoDecimals
_u→unit
_i→index
_b→boolean

All 97 KPIs have been validated against these rules. 0 violations found in the current dataset.

📖 How to Use
1
Start with the KPI Pivot tab for lookup
Use the search box to find any KPI by key or name. The version prefix on each row (e.g., WP/BDGT/FCST) tells you exactly which versions it belongs to.
SearchFilter by formatSort columnsExport CSV
2
Use filters to narrow down your context
Filter by number format ($R, $C, %, U, I, B) to see all KPIs of a specific type. Tick version checkboxes to show only KPIs present in the versions you're working on. Toggle Deviations only to see non-universal KPIs.
3
Check the Sanity Check tab to understand deviations
The matrix groups KPIs by their deviation pattern. Green sections are healthy (all 6 versions). Purple/warning sections show intentional gaps — these are by design and should be respected during configuration.
LY=LLY checkBDGT=FCST checkType validation
4
Refresh when the source sheet changes
The data source is a Google Sheet maintained by the product team. Use the Refresh button below to open it. When the sheet is updated, re-generate this artifact using the Python script in the repo.
